### ACIM-305 — Listen Minute list renders all 25 cached episodes descending
- **Status:** PASS
- **Commit:** fb0b464
- **Diagnosis:**
  - Diagnostic step 1 (server baseline): `curl -s https://www.acimdailyminute.org/podcast-minute.xml | grep -c '<item>'` returned 25 ✓
  - All 25 items have valid RFC2822 pubDates (`EEE, dd MMM yyyy HH:mm:ss GMT`), unique enclosure URLs, and non-empty audio paths
  - PodcastXMLParser correctly parses all items: resets state on `<item>` start, appends on `</item>` end, guards on parseable pubDate + non-empty audioURL
  - PodcastService.persist() correctly upserts all items: keyed by `ep.id` (enclosure URL), `@Attribute(.unique)` on `CachedPodcastEpisode.id`
  - @Query has no fetchLimit, predicate `channel == "minute"` matches persist channel `"minute"`, sort by publishedAt descending ✓
  - **Root cause found in `ListenView.reload(force:)`:** The early-return guard `if hasCache && !force { return }` skipped fetching entirely whenever ANY cached data existed, regardless of session freshness. On a fresh install the feed fetches correctly (cache empty → no early return → 25 episodes). But once even 1 episode was cached (from sprint I's early testing when the feed had fewer items), every subsequent non-forced reload returned immediately without re-fetching, leaving the stale single-row cache permanently displayed
  - Diagnostic step 2 (stale cache test): Not needed — code analysis proves fresh install returns all 25 items and the early-return is the root cause
- **Fix:** Modified `reload(force:)` in ListenView.swift (lines 219-225):
  - Old: `if hasCache && !force { loadState = .loaded; hasLoadedOnce = true; return }`
  - New: (1) `if hasCache { loadState = .loaded }` — show cached data immediately; (2) `if hasCache && hasLoadedOnce && !force { return }` — only skip re-fetch if already fetched THIS session
  - Effect: On every fresh app launch, `hasLoadedOnce` is false (reset with @State), so the fetch always fires, overwriting stale cache with fresh data. Within the same session, tab switches don't re-fetch (optimization preserved). Pull-to-refresh always fetches (force=true bypasses guard)
  - No UserDefaults one-time migration needed — the reload logic fix is cleaner and directly addresses the root cause
  - No changes to PodcastService.swift or FeedService.swift — parser, persist, and query logic all correct
- **Verification:**
  - ./build.sh exit 0, grep 'error:' count = 0
  - No new print() statements vs sprint I baseline (diff clean)
